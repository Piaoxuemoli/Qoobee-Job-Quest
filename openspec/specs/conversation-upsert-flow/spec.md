## Purpose

定义新增与修改投递记录的对话式 upsert 流程，保证采集完整性与更新可追踪性。

## Requirements

### Requirement: 新增记录对话触发与字段采集
系统 SHALL 在用户触发新增投递记录时，按顺序采集公司名、投递类型、当前进度、岗位、官网网址、当前时间，并在缺失时补问。

#### Scenario: 新增流程完整采集
- **WHEN** 用户表达新增投递记录意图
- **THEN** 系统 MUST 进入新增流程并完成必填字段采集后才允许保存
- **AND** 保存后系统 MUST 同步重算 README 统计、明细与时间轴

#### Scenario: 部分字段已在用户输入中提供
- **WHEN** 用户首条消息已包含部分字段
- **THEN** 系统 SHALL 仅补问缺失字段并复述确认后保存

### Requirement: 修改记录对话触发与更新确认
系统 MUST 在用户触发修改记录时先完成记录定位，再采集变更字段，并输出修改前后摘要。

#### Scenario: 修改流程按字段更新
- **WHEN** 用户要求修改某条记录的进度和时间
- **THEN** 系统 SHALL 仅更新指定字段并保持未提及字段不变
- **AND** 更新后系统 MUST 同步重算 README 统计、明细与时间轴

#### Scenario: 定位到多条候选记录
- **WHEN** 用户提供的信息匹配到多条记录
- **THEN** 系统 MUST 要求用户二次确认目标记录后再更新
