## Purpose

定义投递记录的数据结构、必填字段与定位规则，确保新增/修改记录时数据一致且可唯一定位。

## Requirements

### Requirement: 投递记录字段完整性与格式校验
系统 MUST 在新增或修改投递记录时校验标准字段，至少包含 ID、时间、公司、类型、岗位、进度、官网链接。

#### Scenario: 新增记录字段齐全
- **WHEN** 用户触发新增投递记录并完成信息填写
- **THEN** 系统 SHALL 仅在必填字段完整且格式合法时写入记录

#### Scenario: 字段格式不合法
- **WHEN** 用户输入无效时间格式、无效 URL 或空公司名
- **THEN** 系统 SHALL 提示具体字段错误并要求修正后再保存

### Requirement: 记录唯一标识与定位优先级
系统 SHALL 为每条投递记录维护唯一 ID，并在修改流程中优先使用 ID 定位记录。

#### Scenario: 使用 ID 定位记录
- **WHEN** 用户指定记录 ID 发起修改
- **THEN** 系统 MUST 精确匹配该记录并进入字段更新流程

#### Scenario: 未提供 ID 的兜底定位
- **WHEN** 用户未提供 ID 但提供公司、岗位和时间
- **THEN** 系统 SHALL 使用公司+岗位+时间进行兜底匹配并在歧义时要求用户确认
